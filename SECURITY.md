# Security Policy

This repository contains public profile material only. It should never contain secrets, private prompts, access tokens, or unpublished project data.

## Reporting

- Prefer GitHub private vulnerability reporting if it is enabled for this repository.
- If private reporting is not available, open a minimal issue requesting contact and do not include secrets, payloads, private URLs, or personal data.
- Do not post API keys, session tokens, or local filesystem paths in issues or pull requests.
